# mean-project

Project: MechanicStream CRM
Overview
A specialized Customer Relationship Management (CRM) and Invoice Management system designed for independent automotive technicians and self-employed mechanics. This tool replaces manual paper receipts and fragmented digital templates with a centralized, local-first web application.

The Problem
Small-scale mechanics often rely on physical receipt books or generic spreadsheets, making it difficult to track customer history, manage pending job estimates, and look up vehicle-specific data (VIN/Parts) quickly.

The Solution (MVP Features)
Customer Management: A searchable database for client contact information and service history.

Vehicle Tracking: Dedicated records for vehicle specifications (Make, Model, Year, VIN).

Work Order Pipeline: A digital workflow to move jobs from "Price Estimate" to "In Progress" to "Completed/Invoiced."

Local-First Architecture: Designed to run on-premises to ensure maximum data privacy and security for sensitive customer information.

Technical Stack (MEAN)
Database: MongoDB (NoSQL) for flexible service record storage.

Backend: Node.js & Express (RESTful API logic).

Frontend: Angular (Dynamic, responsive dashboard).

Language: TypeScript/JavaScript.
